# Changelog Skill

Format specification for CHANGELOG.md based on Keep a Changelog 1.1.0.

## What This Skill Covers

- **Format Structure**: File header, version sections, change types
- **Types of Changes**: Added, Changed, Deprecated, Removed, Fixed, Security
- **Best Practices**: ISO 8601 dates, comparison links, Unreleased section
- **Bad Practices**: Commit dumps, ignored deprecations, inconsistent updates
- **Conventional Commits Integration**: Mapping commits to changelog sections
- **Workflow**: Pre-commit checklist, release process

## When to Use

Activate this skill when you need to:

- Create a new CHANGELOG.md file
- Add entries to existing changelog
- Review changelog format for correctness
- Prepare changelog for release
