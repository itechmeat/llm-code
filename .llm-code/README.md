# .llm-code

Shared LLM assets for use across multiple projects and tools.

## Quick Start

1. Copy `.llm-code/` folder to your project root
2. Run the link script:

```bash
.llm-code/link-assets.sh
```

3. Add symlinks to `.gitignore` (see below)

## Structure

```
.llm-code/
├── link-assets.sh    # Symlink generator script
├── link-config.sh    # Tool mappings configuration
├── README.md         # This file
└── skills/           # Agent Skills (agentskills.io)
```

## Usage

```bash
# Link all configured tools
.llm-code/link-assets.sh

# Link only specific tools
.llm-code/link-assets.sh -t claude
.llm-code/link-assets.sh -t claude -t github

# Preview changes without making them
.llm-code/link-assets.sh --dry-run

# Show configuration
.llm-code/link-assets.sh --list
```

### Removing Symlinks

To remove a symlink for a specific tool:

```bash
# Remove Claude Code symlink
rm .claude/skills
```

**Note:** `rm` safely removes symlinks without affecting the original `.llm-code/skills/` directory.

### Options

| Option            | Description                                      |
| ----------------- | ------------------------------------------------ |
| `-h, --help`      | Show help message                                |
| `-n, --dry-run`   | Preview changes without making them              |
| `-v, --verbose`   | Verbose output                                   |
| `-l, --list`      | List configured mappings and available tools     |
| `-t, --tool TOOL` | Only process mappings for TOOL (can be repeated) |

## .gitignore

Add symlinks to your project's `.gitignore`:

```gitignore
# LLM tool symlinks (generated by .llm-code/link-assets.sh)
.claude/skills
.github/skills
.opencode/skill
.cursor/skills
```

## Configuration

### Adding a New Tool

Edit `link-config.sh`:

```bash
MAPPINGS=(
    # ... existing ...
    ".newtool:skills:skills"
)
```

Format: `"parent_dir:target_name:asset_dir"`

### Tool-Specific Naming

Some tools use different naming conventions:

```bash
# OpenCode uses "skill" (singular)
".opencode:skill:skills"
```

## Project-Specific Assets

Symlinks only replace shared asset directories. Other files remain project-specific:

```
.github/
├── copilot-instructions.md  # Project-specific (real file)
├── agents/                  # Project-specific (real files)
└── skills -> .llm-code/skills  # Shared (symlink)
```

## Language

All assets MUST be written in English.
